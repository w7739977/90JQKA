## 游戏规则

### 牌组

共 **20 张牌**，4 花色（♠♥♣♦）× 5 点数（10/J/Q/K/A）。

### 流程

1. 房主创建房间，其他玩家通过邀请链接、二维码或房间号加入
2. 全体玩家点击「准备」，房主点击「开始游戏」
3. 由房主开始，**顺时针**轮流摸牌
4. 每张牌摸出后立即触发对应效果
5. **第 4 张 A** 被摸出时游戏结束

### 牌效

| 牌 | 效果 | 说明 |
|----|------|------|
| **10** | 🔄 方向反转 | 摸牌顺序从顺时针变为逆时针（或反之） |
| **J** | 🍺 往公杯加酒 | 摸到 J 的玩家**必须**选择往公杯加 1/2/3 杯酒 |
| **Q** | 🛡️ 获得跳过能力 | 下次轮到自己时可选择跳过；**场上有活跃 Q 时再摸 Q，两张 Q 同时失效** |
| **K** | 😵 全局递增喝酒 | 第 1 张 K 喝 1 杯，第 2 张 K 喝 2 杯，第 3 张 3 杯，第 4 张 4 杯 |
| **A** | 🔁 继续摸牌 | 立即再摸一张；**第 4 张 A 结束游戏**，摸到者喝完公杯所有酒 |

### Q 跳过的细节

- 摸到 Q → 获得跳过能力（头像旁显示 🛡️）
- 轮到自己时有两种选择：**摸牌**（Q 不消耗，保留到以后）或**跳过**（Q 消耗）
- 若场上有活跃 Q 时有人摸到 Q → 两张 Q **同时失效**
- 场上最多只有 **1 个**活跃 Q

### 公杯

- 页面正中显示公杯，带液面动画
- 摸到 J 的玩家选择加 1/2/3 杯，公杯液面上升
- 第 4 张 A 结束时，摸到者喝完公杯所有酒

### 结算

- 游戏结束时显示喝酒排行榜
- 喝酒数**跨局保留**，直到房间销毁
- 房主可选择「再来一局」

### 观战机制

- 游戏进行中，新玩家加入房间后自动标记为**观战**状态
- 观战玩家不能摸牌、不能跳过，轮转时自动跳过
- 一局结束后，观战标记自动清除，下一局正常参与
- 页面显示观战横幅和标签提示

## 技术栈

- **后端**: Node.js + Express + Socket.IO
- **前端**: 原生 HTML / CSS / JavaScript（SPA，Hash 路由）
- **状态管理**: 服务端内存 `Map`，客户端 `localStorage`
- **实时通信**: Socket.IO


## API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/createRoom` | POST | 创建房间 |
| `/api/joinRoom` | POST | 加入房间 |
| `/api/getRoom` | POST | 获取房间状态 |
| `/api/ready` | POST | 准备/取消准备 |
| `/api/startGame` | POST | 开始游戏（房主，全员准备后） |
| `/api/drawCard` | POST | 摸牌（当前玩家） |
| `/api/addWine` | POST | 加酒（摸到 J 后，1-3 杯） |
| `/api/skipTurn` | POST | 跳过回合（使用 Q） |
| `/api/restartGame` | POST | 再来一局（房主） |
| `/api/kickPlayer` | POST | 踢人（房主） |

## 服务端数据模型

```
Room:
  roomId, ownerOpenId, status (waiting|playing|finished),
  deck[20], drawIndex, currentPlayerIdx, direction (1|-1),
  publicCup, kCount, aCount, players[], turnLog[], pendingAction

Player:
  openId, nickName, avatarUrl, ready, drinks,
  activeQ, drawnCards[], offline, lastDrawnCard

PendingAction:
  type: 'addWine', playerIdx, card
```

## 状态流转

```
waiting ──房主开始(all ready)──→ playing ──第4张A──→ finished
   ↑                                                  |
   +──────────── 房主点"再来一局" (restartGame) ←──────+
```

## 部署排障

| 现象 | 原因 | 解决 |
|------|------|------|
| `EADDRINUSE :::3001` | 3001 端口被占用 | `lsof -i:3001` 找到占用进程，停掉后重启 |
| 重启次数异常高 | 进程启动即崩溃 | `pm2 logs --err` 查看错误日志 |
| `git pull` 后无变化 | pm2 未重启 | 必须 `pm2 restart 90jqka` |
| 部署后页面缓存 | 浏览器缓存旧 JS | 强制刷新（Ctrl+Shift+R） |

## 与诈金花 Web 版的关系

本项目基于 [pj_zhajinhua_web](https://github.com/w7739977/zhajinhua-web) 架构开发，复用了以下设计：

- 房间管理系统（创建/加入/邀请/踢人）
- Socket.IO 实时通信机制
- 在线状态追踪与离线清理
- 邀请链接 + 二维码生成
- 测试框架（test-runner / sim-runner）
- 深色牌桌视觉主题
- Hash 路由 SPA 架构

核心差异：游戏引擎、牌组规则、摸牌机制、公杯系统均为 90JQKA 独有设计。
